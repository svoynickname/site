Эта статья поможет вам быстро и бесплатно создать веб-сайт для размещения портфолио, резюме или блога.

Учётная запись Google

Для размещения сайта нам потребуется учетная запись Google. Если у вас есть почта на Gmail, значит она у вас есть. Если нет, создайте учётную запись.

Домен

Это необязательно. В мире существуют разные провайдеры доменных имен, найти можно тут Google.

Генератор статических сайтов Hugo

Hugo - это инструмент на основе языка Go , который генерирует статические веб-сайты. Вы можете использовать различные шаблоны и создавать разные типы сайтов, такие как блоги, портфолио и т. д. Скачайте его отсюда.

Firebase

Firebase - это платформа для мобильных устройств и веб-приложений, приобретенная Google несколько лет назад. Firebase предлагает хостинг в качестве одной из своих функций. Однако многие разработчики мобильных приложений используют его для аналитики, уведомлений, отчетов о сбоях приложений. Мы буде использовать его для размещения нашего сайта.

Node.js

Node.js - это среда выполнения JavaScript с открытым исходным кодом. Его необходимо установить на свой компьютер, чтобы работали инструменты Firebase. Вы можете скачать и установить его здесь.

Шаг 1: Установите Hugo на свой компьютер

Windows: вы получите простой переносимый исполняемый файл. Вы можете разместить его где угодно и запустить через командную строку. Вы можете добавить его к своей переменной пути в переменных среды Windows, чтобы ссылаться на него где угодно.

Mac: вы можете установить его, используя Homebrew. Если у вас не установлен brew, вы можете скачать здесь .

В любом случае, убедитесь, что вы можете получить доступ к Hugo, дав команду ниже.

Шаг 2: Создайте шаблон сайта

Перейдите к месту, где вы решили создать свой веб-сайт, и введите команду ниже:

$ hugo new site <path_to_folder>

В данном месте вы можете увидеть структуру папок, как показано на рисунке ниже. Изображение взято на Windows 10

Эти папки являются просто заполнителями для вашего контента. Весь текстовый контент вашего сайта попадает в папку содержимого .

Вы можете запустить команды ниже, чтобы добавить новые файлы.

$ hugo new about.md

Если вы хотите добавить запись в блоге, создайте папку с именем «blog» в папке «content» и начните добавлять свои файлы. Эти файлы имеют расширение «.md», которые являются файлами Markdown .

Markdown - это язык разметки в формате простого текста. Это красиво и легко. Есть множество руководств о Markdown, вот одно из них.

Шаг 3: Установите тему для сайта

У Hugo отличное сообщество. Здесь вы найдете разнообразные темы. Выберите тему, которая соответствует вашим требованиям.

Я выбрал тему Introduction. Нажав на кнопку загрузки, вы будете перенаправлены на GitHub.

У каждой темы будет свой способ настройки. Эта конкретная тема не имеет много шагов. Просто клонируйте или загрузите zip в папку с темами. Скопируйте файл config.toml в корневую папку вашего сайта.

Шаг 4: Настройте ваши предпочтения

Откройте файл config.toml и начните его редактирование. Укажите свое имя и другие данные, которые вы хотите показать на сайте. Некоторые темы поддерживают Google Analytics, поэтому вы можете отслеживать количество посещений пользователей и т. Д. Вы можете указать свой идентификатор GA для сбора данных.

Шаг 5: Настройте проект хостинга Firebase

Как я упоминал ранее, Firebase - это красивая мобильная платформа с множеством функций. Я использовал хостинг Firebase для размещения своего статического сайта, созданного с помощью Hugo.

Для использования сервисов Firebase используйте вашу учетную запись Google и войдите на сайт Firebase .

Нажмите «Перейти к консоли». Создайте проект, дав ему имя. Вам будет показана страница обзора, на которой вы должны выбрать « начало работы на хостинге».

Шаг 6: Настройте инструменты Firebase на своем компьютере

Откройте свой терминал / интерфейс командной строки на своем компьютере и введите команду ниже.

$ npm install -g firebase-tools

Приведенная выше команда устанавливает пакет Firebase-tools. Вам нужно выполнить еще несколько команд, чтобы иметь возможность развернуть ваш сайт напрямую.

$ firebase login

Эта команда соединяет ваш компьютер с проектом Firebase. Это позволяет вам перечислить и выбрать проект, в который вы хотите внести свои изменения.

$ firebase list

Вы можете запустить команду выше, чтобы увидеть проект, который вы создали. В завершение всего рабочего процесса нам нужно инициализировать корневую папку вашего сайта как корневую папку проекта Firebase.

$ firebase init

Он задаст вам несколько вопросов, таких как

Какие функции Firebase CLI вы хотите настроить? Ответ: хостинг.
Выберите проект Firebase по умолчанию для этого каталога. Ответ: Выберите проект Firebase, созданный на шаге 5.
Вы хотите использовать в качестве публичного каталога? Ответ: да.
Настроить как одностраничное приложение? Ответ: да.

Чтобы избежать путаницы, у меня есть подробные скриншоты, сделанные на каждом этапе для вашей справки. Его можно скачать здесь .

В заключение! Инициализация Firebase завершена.

Шаг 7: Проверьте свой сайт локально

Запустите команду ниже, чтобы проверить ваш сайт локально на вашем компьютере.

$ hugo server -w

Hugo поставляется с легким высокопроизводительным веб-сервером, где вы можете проверить все свои изменения. Убедитесь, что все ваши изображения размещены в папке static / img . В итеративном процессе измените ваш config.toml и проверьте их в браузере. Ниже указан порт, на котором будет работать ваш сервер.

http://localhost:1313

Шаг 8: Разверните свой сайт

Введите ниже команду для создания вашего сайта и отправьте его в соответствующий проект Firebase, который вы настроили на шаге 5 .

$ hugo && firebase deploy

Шаг 9: подключите его к своему домену (необязательно)

Firebase предоставляет возможность подключить ваш домен к приложению Firebase. Нажмите на подключить домен и укажите свой домен и добавьте записи TXT, чтобы подтвердить право собственности на домен.

Для этого войдите на портал регистратора вашего домена.

Шаг 10: Создать блог

Создать пост в блоге довольно просто. Hugo понимает файлы уценки. Просто зайдите в папку content-> blog (расположение папки зависит от темы). Создайте файл уценки. Повторите Шаг 8, чтобы увидеть результаты.
